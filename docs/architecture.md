# NatChain 架构设计

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      NatChain 区块链                         │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────────────┐ │
│  │ x/bank  │  │x/miner  │  │x/wallet │  │  x/merchant     │ │
│  │ (SDK)   │  │ (自定义) │  │ (自定义) │  │   (自定义)      │ │
│  └─────────┘  └─────────┘  └─────────┘  └─────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│                     Cosmos SDK 核心                          │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐    │
│  │ Auth   │ │ Staking│ │  Bank  │ │Distribution│ │Gov│    │
│  └────────┘ └────────┘ └────────┘ └────────┘ └────────┘    │
├─────────────────────────────────────────────────────────────┤
│                     CometBFT (共识层)                        │
├─────────────────────────────────────────────────────────────┤
│                        P2P 网络                              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                     外部服务层                               │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │ REST API    │  │  gRPC API   │  │  商家回调服务       │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## 模块职责

| 模块 | 类型 | 职责 |
|------|------|------|
| x/auth | SDK | 账户认证、签名验证 |
| x/bank | SDK | 代币转账 |
| x/staking | SDK | 验证者质押（矿机绑定） |
| x/distribution | SDK | 奖励分配 |
| x/gov | SDK | 链上治理 |
| **x/miner** | 自定义 | 矿机注册、绑定、管理 |
| **x/wallet** | 自定义 | 安全额度、自动转账、伙伴管理 |
| **x/merchant** | 自定义 | API Key、支付接口、回调 |

```

┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│ 矿机   │───▶│Miner模块│───▶│Staking │───▶│ 出块奖励│
│ (CPU)  │    │(验证绑定)│    │(验证者) │    │(分配)  │
└────────┘    └────────┘    └────────┘    └────────┘

支付流程：
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐

└────────┘    └────────┘    └────────┘    └────────┘
```

## 共识机制

- **共识引擎**: CometBFT (Tendermint)
- **出块时间**: 5 秒
- **区块大小**: 动态调整
- **验证者数量**: 最少 4 个，最多 100 个
- **矿机限制**: 1000 台固定上限

## 安全设计

| 层面 | 措施 |
|------|------|
| 网络 | TLS、节点认证 |
| 共识 | CometBFT BFT 共识 |
| 交易 | 签名验证、重放保护 |
| 钱包 | 安全额度、自动转账 |
| API | HMAC 签名、IP 白名单 |
